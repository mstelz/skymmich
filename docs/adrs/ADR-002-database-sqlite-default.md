# ADR-002: SQLite as Default Database

**Status:** Active
**Area:** database
**Date:** 2026-04-09

---

## Context

Skymmich is a self-hosted application targeting individual astrophotographers. The typical deployment is a single Docker container or local install serving one user. Requiring a PostgreSQL server for basic functionality would add setup friction and infrastructure overhead for users who just want to manage their images.

## Decision

SQLite is the default database, used when no `DATABASE_URL` environment variable is set. It stores data in a local `local.db` file with zero configuration. PostgreSQL is supported as an opt-in alternative for users who want external database management, but is not required.

SQLite auto-applies Drizzle migrations on startup. PostgreSQL migrations are a manual step via a separate script.

## Consequences

### Enables
- Zero-configuration startup: users run the app and it works immediately
- Simple Docker deployment: no database container or network configuration needed
- Easy backup: the entire database is a single file that can be downloaded from the admin panel

### Costs
- SQLite has limited concurrent write support — not an issue for single-user but could be if the app ever supports multiple users
- Some SQL features work differently (JSON arrays, full-text search), requiring dialect-conditional code
- PostgreSQL users don't get auto-migration on startup, creating an inconsistent experience

---

## Rules

- **DO:** Ensure all new features work with SQLite first, as it is the default path
- **DO:** Provide a migration path when schema changes require manual PostgreSQL intervention
- **DON'T:** Use PostgreSQL-only features (arrays, advanced JSON operators) without a SQLite equivalent
- **WHEN IN DOUBT:** Optimize for the SQLite user experience — that's the majority of deployments

> **Scope:** Database initialization in `apps/server/src/db.ts`, migration scripts in `tools/`, and all storage layer code

## Evidence

- `.env.example`: `DATABASE_URL` is commented out with explanation that SQLite is the default

- `tools/migrations/sqlite/` contains 9 migration files; PostgreSQL migrations are handled separately
