# ADR-008: Native Fetch for All HTTP Communication

**Status:** Active
**Area:** client
**Date:** 2026-04-09

---

## Context

The application was migrated from Express/Socket.IO/axios to Hono/WebSocket/fetch in v0.8.0. This migration eliminated the dependency on axios for HTTP client calls. The native `fetch` API is available in both the browser and Node.js (since Node 18+), making a third-party HTTP client unnecessary.

## Decision

All HTTP communication uses the native `fetch` API — both client-to-server requests in the frontend and server-to-external-service requests in the backend (Immich API, astrometry.net API, OpenNGC GitHub downloads, CDS survey images). No HTTP client library (axios, got, ky, etc.) is used anywhere in the codebase.

On the client side, `apiRequest()` in `queryClient.ts` wraps `fetch` with JSON content-type headers and error handling. On the server side, `fetch` is called directly with manual timeout via `AbortSignal.timeout()`.

## Consequences

### Enables
- Zero HTTP client dependencies — one fewer thing to update, audit, and bundle
- Consistent API across client and server code
- Leverages platform-native functionality that improves with each Node.js release

### Costs
- No built-in request/response interceptors (must be implemented manually if needed)
- No automatic retry, timeout, or circuit-breaker behavior — must be added per-call
- Error handling requires manual `.ok` checking and `.text()` extraction (no automatic error wrapping)

---

## Rules

- **DO:** Use native `fetch` for all HTTP requests (client and server)
- **DO:** Use `AbortSignal.timeout()` for server-side requests to external services
- **DO:** Check `response.ok` and handle errors explicitly
- **DON'T:** Add axios, got, ky, or other HTTP client libraries
- **DON'T:** Assume `fetch` throws on non-2xx responses — it doesn't
- **WHEN IN DOUBT:** Follow the `apiRequest()` pattern for client requests; use raw `fetch` with explicit error handling for server-side requests

> **Scope:** All HTTP calls in `apps/client/src/` and `apps/server/src/`
