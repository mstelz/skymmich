# ADR-005: Module-Level Singletons for Services

**Status:** Active
**Area:** services
**Date:** 2026-04-09

---

## Context

The backend services (storage, config, catalog, astrometry, worker manager, cron manager, immich sync, XMP sidecar) all need to be instantiated once and shared across route handlers. The application is single-user and single-process (aside from the worker), so complex dependency injection is not warranted.

## Decision

Every service is instantiated as a module-level `export const` singleton at the bottom of its file (e.g., `export const storage = new DbStorage()`). Route handlers and other services import these singletons directly. There is no dependency injection container, no factory pattern, and no interface-based abstraction.

This pattern emerged organically rather than as a deliberate architectural choice.

## Consequences

### Enables
- Simple, zero-ceremony service access — just import and use
- No DI framework overhead or configuration
- Clear, predictable initialization order (module evaluation order)

### Costs
- Services are tightly coupled to their implementations — swapping implementations (e.g., for testing) requires mocking the module
- Circular dependency risk: if service A imports service B and vice versa, module initialization can fail (the `immich-sync` module uses dynamic `import()` to work around this)
- No lifecycle management — singletons are created on module import, not on application start

---

## Rules

- **DO:** Export services as `export const serviceName = new ServiceClass()` at module bottom
- **DO:** Use dynamic `import()` to break circular dependencies between services
- **DON'T:** Create multiple instances of a service — always import the singleton
- **DON'T:** Pass service instances as constructor parameters to other services (import the singleton instead)
- **WHEN IN DOUBT:** Follow the existing pattern — create the singleton in the service file, import it where needed

> **Scope:** All service files in `apps/server/src/services/`

## Evidence

- `apps/server/src/services/storage.ts`: `export const storage = new DbStorage()`
- `apps/server/src/services/config.ts`: `export const configService = new ConfigService()`
- `apps/server/src/services/catalog.ts`: `export const catalogService = new CatalogService()`
- `apps/server/src/services/worker-manager.ts`: `export const workerManager = new WorkerManager()`
- `apps/server/src/services/cron-manager.ts`: `export const cronManager = new CronManager()`
