# SPEC-018: Location Management

**Status:** Active
**Area:** locations
**Date:** 2026-02-27

---

## Purpose

This feature allows astrophotographers to save and manage their observation locations. Each location stores GPS coordinates, elevation, Bortle class (light pollution rating), and notes. Locations can be selected when editing image metadata to associate a capture with a specific observing site.

---

## Behavior

Users manage locations through a dedicated page with a list view and an editor modal that includes an interactive Leaflet map for coordinate selection.

### Entry Points

- `/locations` — Locations management page
- `GET /api/locations` — List all locations
- `GET /api/locations/:id` — Get a single location
- `POST /api/locations` — Create a location
- `PATCH /api/locations/:id` — Update a location
- `DELETE /api/locations/:id` — Delete a location

### Key Flows

1. User opens the location editor modal
2. A Leaflet map with OpenStreetMap tiles provides an interactive picker for selecting coordinates
3. User clicks or drags a marker to set latitude/longitude
4. User fills in: name, Bortle class (1-9 scale), elevation, and optional notes
5. Location is saved to the database
6. When editing an image's location, the user can select from saved locations or set coordinates manually

---

## Patterns Used

- Leaflet + React Leaflet for interactive map rendering
- Modal-based editing (reused `BaseModal` pattern)
- Standard REST CRUD

---

## Business Rules

- **Coordinate precision:** Latitude and longitude are stored as floating-point values with full precision.
- **Bortle class:** An integer from 1 (darkest) to 9 (brightest), representing the light pollution level at the site.

---

## Edge Cases & Constraints

- The Leaflet map requires internet connectivity to load OpenStreetMap tiles
- Deleting a location does not update images that reference that location's coordinates

---

## Known Limitations

- Locations are not directly linked to images via foreign key — they serve as a reference for manual coordinate entry
- No weather or sky quality data integration
- No distance calculation or "nearest location" feature
- No import from GPS tracks or external sources
