# SPEC-004: Immich Library Sync

**Status:** Active
**Area:** immich-integration
**Date:** 2025-07-10

---

## Purpose

This feature imports astrophotography images from a self-hosted Immich photo management instance into Skymmich. It is the primary (and currently only) way to ingest images into the system. It also supports bidirectional metadata sync — pushing enriched Skymmich data (plate-solving results, equipment, descriptions, coordinates, tags) back to Immich.

---

## Behavior

The sync process fetches image assets from Immich, creates corresponding records in the Skymmich database with extracted EXIF metadata, and manages the lifecycle of synced images (adding new ones, removing deleted ones). A scheduled cron job can automate this process.

### Entry Points

- `POST /api/immich/sync-immich` — Trigger a manual sync
- `POST /api/immich/sync-metadata/:imageId` — Push metadata for one image back to Immich
- `POST /api/immich/sync-metadata-all` — Push metadata for all images back to Immich
- `POST /api/immich/test-immich-connection` — Test connectivity to the Immich server
- `POST /api/immich/albums` — Fetch available Immich albums for selection
- Cron job: `immich-sync` (configurable schedule, default every 4 hours)

### Key Flows

**Ingest (Immich -> Skymmich):**
1. Fetch assets from Immich — either from selected albums (if album sync is enabled) or from the full library via paginated metadata search
2. De-duplicate assets by Immich ID (important when assets appear in multiple albums)
3. For each new asset not already in the database, extract EXIF data (focal length, aperture, ISO, exposure time, camera, lens, GPS coordinates) and create an AstroImage record
4. Thumbnail and full URLs are stored as proxy paths (`/api/assets/{immichId}/thumbnail`)
5. Remove images from Skymmich whose Immich asset IDs no longer exist in the source
6. Broadcast sync completion via WebSocket with counts

**Metadata writeback (Skymmich -> Immich):**
1. Update native Immich asset fields (description, GPS coordinates) via `PUT /api/assets/{id}`
2. Write custom astronomy metadata (RA, Dec, pixel scale, FOV, equipment, acquisition summary) via `PUT /api/assets/{id}/metadata`
3. Sync tags: create Immich tags if they don't exist, then assign them to the asset

## Patterns Used

- Cron-scheduled sync via `node-cron` with configurable frequency
- Dynamic import of `immich-sync` module (lazy-loaded to avoid circular dependencies)
- Paginated API consumption for full-library sync (1000 assets per page)
- WebSocket broadcast on sync completion for real-time UI refresh

---

## Business Rules

- **Album sync requires selected albums:** If `syncByAlbum` is enabled but no albums are selected, the sync throws an error.
- **De-duplication by Immich ID:** An asset that exists in multiple albums is only imported once.
- **Removal of orphaned images:** Images in Skymmich whose `immichId` is no longer found in the Immich source are automatically deleted during sync.
- **Metadata writeback is opt-in:** Each writeback category (description, coordinates, tags) can be independently enabled/disabled in admin settings.
- **API key redaction:** When the admin UI sends back settings with redacted API keys (starting with `••••`), the server preserves the existing stored key.
- **Tag filtering on writeback:** Only relevant tags are synced to Immich — star names from plate solving are excluded using the same filtering as the gallery.

---

## Edge Cases & Constraints

- Full-library sync uses Immich's metadata search endpoint (POST) rather than a simple list endpoint, due to Immich API limitations with large libraries
- Connection testing validates URL protocol (HTTP/HTTPS only) to prevent SSRF
- All new images default to `objectType: 'Deep Sky'` and `tags: ['astrophotography']`
- EXIF `lensModel` maps to the telescope field; `make + model` maps to camera

---

## Known Limitations

- No granular filtering during import — all images from the selected source (library or albums) are imported. Future improvement: filter by Immich tag, folder, or metadata
- No incremental sync — each sync re-checks all assets (though existing images are skipped by Immich ID lookup)
- No way to import images without Immich — this is currently the only ingest path
- Metadata writeback errors for individual images are logged but don't fail the overall operation
