# SPEC-009: Equipment Groups

**Status:** Active
**Area:** equipment
**Date:** 2026-03-11

---

## Purpose

This feature allows users to define named groups of equipment (e.g., "Deep Sky Rig", "Planetary Setup") and apply an entire group to an image in one action. This reduces the repetitive task of individually linking the same set of telescope, camera, mount, and filters to each new image.

---

## Behavior

Users create groups, assign equipment members, and then apply groups to images. Applying a group links all member equipment items to the target image.

### Entry Points

- `GET /api/equipment-groups` — List all groups with members
- `GET /api/equipment-groups/:id` — Get a single group
- `POST /api/equipment-groups` — Create a group with name, description, and member IDs
- `PUT /api/equipment-groups/:id` — Update group name/description
- `DELETE /api/equipment-groups/:id` — Delete a group
- `PUT /api/equipment-groups/:id/members` — Replace group membership (set member IDs)
- `POST /api/equipment-groups/:id/apply/:imageId` — Apply group to an image

### Key Flows

1. User creates a group with a name and selects equipment items as members
2. Group membership is managed as a set — replacing all members at once via PUT
3. Applying a group to an image creates `image_equipment` records for each member that isn't already linked
4. The response includes how many items were actually added (skipping duplicates)

---

## Patterns Used

- Set-replacement for membership (replace all members rather than add/remove individually)
- Idempotent apply: applying a group to an image skips already-linked equipment

---

## Business Rules

- **Name is required:** Group creation requires a name.
- **Member IDs must be an array:** The members endpoint validates that `memberIds` is an array.
- **Apply is additive and idempotent:** Applying a group adds missing equipment links without duplicating existing ones.

---

## Edge Cases & Constraints

- Deleting a group does not affect existing image-equipment relationships that were created by applying the group
- A group with no members is valid but applying it is a no-op

---

## Known Limitations

- No UI indicator of which groups have been applied to an image
- No "unapply" — removing a group's equipment from an image requires individual unlinking
- No ordering of members within a group
