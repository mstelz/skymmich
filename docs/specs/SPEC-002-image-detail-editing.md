# SPEC-002: Image Detail Editing

**Status:** Active
**Area:** image-management
**Date:** 2025-07-12

---

## Purpose

This feature allows users to edit metadata for individual astrophotography images. Users can rename images, add descriptions, assign locations, link equipment, manage acquisition data, assign targets, and sync metadata back to Immich. It provides the detailed per-image management workflow accessed from the image overlay.

---

## Behavior

When a user opens an image overlay, they can access multiple editing modals and inline editors for different aspects of the image's metadata. Changes are persisted via PATCH/PUT requests and trigger React Query cache invalidation.

### Entry Points

- Image overlay (opened from gallery click or deep-link) — provides access to all editing actions
- `PATCH /api/images/:id` — Update image metadata fields
- `GET /api/images/:id` — Fetch latest image data (overlay re-fetches on mount)

### Key Flows

1. **Title editing:** Opens a modal to rename the image's display title. The original filename is preserved and shown as a reference. Title cannot be empty.
2. **Description editing:** Opens a modal to edit a freeform text description of the image.
3. **Location editing:** Opens a modal with a Leaflet map picker to set latitude/longitude/altitude for the image. If the image is linked to Immich and coordinates change, they are automatically synced back to the Immich asset.
4. **Target assignment:** Opens a target picker modal to associate the image with a catalog object (deep-sky target).
5. **Equipment management:** Inline section to link/unlink equipment items to the image with optional settings and notes per relationship.
6. **Acquisition editing:** Inline editor for per-filter sub-exposure data (filter name, frame count, exposure time, gain, offset, binning, sensor temp, date, notes).
7. **Immich metadata sync:** A button to push all Skymmich metadata (description, coordinates, tags, equipment, acquisition summary) back to the Immich asset.

---

## Patterns Used

- Modal-based editing with `BaseModal` component for consistent UI
- React Query cache invalidation on save (`invalidateQueries`)
- Optimistic initial data pattern: overlay uses prop image as `initialData` while re-fetching latest
- Collapsible sections in the overlay for progressive disclosure of metadata

---

## Business Rules

- **Title cannot be empty:** The save button is disabled when the title field is blank or unchanged.
- **Coordinate sync to Immich:** When latitude or longitude is updated on an image with an `immichId`, the coordinates are automatically pushed to Immich via the Immich API. This happens server-side during the PATCH handler.
- **Visual rename only:** Changing the title does not rename the underlying file. The original filename is preserved for system linking.

---

## Edge Cases & Constraints

- The image overlay re-fetches the image on mount to ensure fresh data, using the prop as initial data to avoid a loading flash
- Immich metadata sync can fail silently if the Immich server is unreachable — errors are logged server-side but the user sees an error message in the UI

---

## Known Limitations

- Only the title field has a dedicated editor modal; other image metadata fields (object type, constellation, etc.) are not directly editable through the UI — they come from plate solving or Immich sync
- No undo/revert capability for metadata changes
- No bulk editing across multiple images
