# SPEC-007: XMP Sidecar Generation

**Status:** Active
**Area:** plate-solving
**Date:** 2025-07-10

---

## Purpose

This feature generates XMP sidecar files containing plate-solving results, celestial annotations, equipment data, and image tags. These files provide a standard metadata interchange format that can be consumed by other astrophotography and photo management tools.

---

## Behavior

After a successful plate solve, the system generates an XMP file alongside the image containing structured astronomical metadata. Users can also download sidecars on demand via the API.

### Entry Points

- Automatic: triggered after successful plate solving (if sidecar generation is enabled in admin settings)
- `GET /api/images/:id/sidecar` — Retrieve an existing sidecar file (with optional `?download=true` for attachment disposition)

### Key Flows

1. After plate solving completes, the system checks if sidecar generation is enabled
2. The XMP content is generated with: calibration data (RA, Dec, pixel scale, orientation, field of view), structured annotations (type, names, pixel coordinates, magnitude), machine tags, equipment details (grouped by type), and image metadata
3. The sidecar is written to the configured output path
4. If `organizeByDate` is enabled, sidecars are organized into `yyyy-mm` subdirectories based on the image's capture date
5. The filename is derived from the original image filename with an `.xmp` extension

---

## Patterns Used

- XMP metadata format following Adobe namespace conventions
- Custom `astro:` namespace for astronomy-specific fields
- Configurable output path with environment variable override (`XMP_SIDECAR_PATH`)
- Date-based directory organization

---

## Business Rules

- **Opt-in via admin settings:** Sidecar generation is controlled by the `sidecar.enabled` config flag. Defaults to enabled.
- **Output path precedence:** `XMP_SIDECAR_PATH` environment variable overrides the admin setting, which overrides the default (`./sidecars`).
- **Equipment grouping:** Equipment in the sidecar is organized by type (telescope, camera, mount, filter) for readability.
- **General-purpose format:** The XMP format aims for broad compatibility rather than targeting a specific tool.

---

## Edge Cases & Constraints

- Sidecar path resolution depends on the image having an `originalPath` or a usable filename
- Directory creation is attempted automatically if the output path doesn't exist

---

## Known Limitations

- XMP sidecar writing has not been verified to work correctly in Docker environments — there may be volume mount or file permission issues preventing writes
- No validation that the generated XMP conforms to a specific schema standard
- No mechanism to regenerate sidecars for previously solved images in bulk
- The `astro:` namespace is custom and may not be recognized by third-party tools without configuration
