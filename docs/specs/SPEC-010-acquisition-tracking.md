# SPEC-010: Acquisition Tracking

**Status:** Active
**Area:** equipment
**Date:** 2026-02-27

---

## Purpose

This feature tracks per-filter sub-exposure details for each astrophotography image. Astrophotographers typically capture many sub-exposures through different filters (Luminance, Ha, OIII, SII, RGB, etc.) and need to record the breakdown. This data auto-computes image summary fields like total integration time and frame count.

---

## Behavior

Each image can have multiple acquisition rows, one per filter or exposure configuration. When acquisition data changes, the parent image's summary fields are recomputed.

### Entry Points

- `GET /api/images/:id/acquisitions` — List acquisition entries for an image
- `POST /api/images/:id/acquisitions` — Add an acquisition entry
- `PUT /api/images/:imageId/acquisitions/:id` — Update an acquisition entry
- `DELETE /api/images/:imageId/acquisitions/:id` — Delete an acquisition entry
- Inline acquisition editor in the image overlay UI

### Key Flows

1. User adds acquisition rows specifying: filter name, frame count, exposure time per frame, and optional fields (gain, offset, binning, sensor temp, date, notes)
2. `frameCount` and `exposureTime` are required for each row
3. After any create/update/delete, the image's summary fields are recomputed via `computeImageSummary()`:
   - `frameCount`: sum of all acquisition row frame counts
   - `totalIntegration`: sum of (frameCount * exposureTime) across all rows, converted to hours
   - `filters`: comma-separated unique filter names
   - `exposureTime`: single value if uniform, or "min - max" range if varied
4. If a telescope is linked to the image, `focalLength` and `aperture` are also populated from telescope specifications

---

## Patterns Used

- Pure function for summary computation (`computeImageSummary` in shared package) — no DB access, fully testable
- Summary recomputation is triggered on every acquisition change (not lazy/cached)

---

## Business Rules

- **Required fields:** `frameCount` and `exposureTime` are required per acquisition row. All other fields are optional.
- **Auto-summary:** Image-level summary fields are always derived from acquisition data + equipment, never manually set when acquisition rows exist.
- **Filter name deduplication:** The image's `filters` field contains unique filter names only.
- **Exposure time display:** If all rows have the same exposure time, displayed as a single value (e.g., "300s"). If varied, displayed as a range (e.g., "60s - 300s").

---

## Edge Cases & Constraints

- An image with no acquisition rows retains whatever summary values were set during import (from EXIF)
- Filter ID is stored alongside filter name but is not currently used for lookups
- Deleting all acquisition rows does not reset the image's summary fields to their original EXIF values

---

## Known Limitations

- No validation that frame count and exposure time are positive numbers
- No integration with calibration frames (darks, flats, bias)
- No per-session grouping of acquisition data
