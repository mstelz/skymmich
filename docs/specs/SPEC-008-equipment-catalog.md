# SPEC-008: Equipment Catalog

**Status:** Active
**Area:** equipment
**Date:** 2026-02-27

---

## Purpose

This feature allows astrophotographers to catalog their imaging equipment — telescopes, cameras, mounts, and filters — with type-specific technical specifications. Equipment items can be linked to individual images to track what gear was used for each capture.

---

## Behavior

Users manage a list of equipment items through a dedicated page and can associate equipment with images via the image overlay. Each equipment type has its own set of specification fields.

### Entry Points

- `/equipment` — Equipment management page
- `GET /api/equipment` — List all equipment
- `POST /api/equipment` — Create equipment
- `PUT /api/equipment/:id` — Update equipment
- `DELETE /api/equipment/:id` — Delete equipment
- `GET /api/images/:id/equipment` — Get equipment linked to an image
- `POST /api/images/:id/equipment` — Link equipment to an image
- `PUT /api/images/:imageId/equipment/:equipmentId` — Update link settings/notes
- `DELETE /api/images/:imageId/equipment/:equipmentId` — Unlink equipment from an image

### Key Flows

1. User creates equipment with a name, type, optional description, cost, acquisition date, and type-specific specifications
2. Type-specific specification fields are defined in `EQUIPMENT_SPEC_FIELDS` (shared between client and server)
3. Specifications are stored as a JSON object in the `specifications` column
4. Equipment can be linked to images with optional per-image settings and notes
5. When equipment is linked to an image, the image's summary fields (focal length, aperture) can be auto-computed from telescope specifications

---

## Patterns Used

- Type-discriminated specification fields: each equipment type (telescope, camera, mount, filter) has a different set of editable fields defined in shared types
- JSON column for flexible specifications storage
- Join table (`image_equipment`) with optional `settings` and `notes` for per-image context

---

## Business Rules

- **Name and type are required:** Equipment creation and updates validate that both fields are present.
- **Type-specific specs:** Telescope specs include focal length, aperture, and focal ratio. Camera specs include sensor size, pixel size, and resolution. Mount and filter types have their own relevant fields.
- **Image summary auto-computation:** When acquisition data or equipment changes, image summary fields (total integration, frame count, focal length from telescope) are recomputed from the linked data.

---

## Edge Cases & Constraints

- Deleting equipment does not cascade to image-equipment relationships — the relationship records may become orphaned
- Equipment cost and acquisition date are optional tracking fields with no business logic attached

---

## Known Limitations

- No image upload for equipment photos (the `imageUrl` field exists but has no upload mechanism)
- No equipment utilization statistics (e.g., "used in N images, X hours total")
- No import/export of equipment data
