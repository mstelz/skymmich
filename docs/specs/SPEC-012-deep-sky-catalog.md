# SPEC-012: Deep-Sky Catalog (OpenNGC)

**Status:** Active
**Area:** sky-and-targets
**Date:** 2026-03-20

---

## Purpose

This feature provides a searchable database of deep-sky objects (galaxies, nebulae, clusters, etc.) from the OpenNGC catalog. It enriches image metadata with object information, powers the target-matching system, supplies thumbnails from sky survey data, and enables users to browse and search the catalog for observation planning.

---

## Behavior

The catalog is loaded from GitHub on first startup and persisted in the database. It provides search, browse, and lookup capabilities, and is used internally for auto-matching plate-solving tags to known objects.

### Entry Points

- `GET /api/catalog/browse` — Paginated catalog browsing with filtering and sorting
- `GET /api/catalog/search` — Autocomplete search by name/common name/identifier
- `GET /api/catalog/:name` — Lookup a single object by name
- `GET /api/catalog/status` — Check catalog load status (object count)
- `GET /api/catalog/thumbnail/:name` — Fetch or serve cached survey thumbnail for an object
- `POST /api/catalog/load` — Manually trigger catalog reload from GitHub
- `POST /api/catalog/check-updates` — Check if the upstream OpenNGC data has been updated
- `POST /api/catalog/backfill-targets` — Re-match all images without a target name against the catalog

### Key Flows

**Catalog loading:**
1. On server startup, check if catalog has data (`getStatus()`)
2. If empty, fetch NGC.csv and addendum.csv from the OpenNGC GitHub repository
3. Parse semicolon-delimited CSV with quoted field handling
4. Normalize object names (e.g., "NGC0001" -> "NGC 1")
5. Convert RA/Dec from sexagesimal to decimal degrees
6. Deduplicate (prefer NGC entry over addendum) and bulk insert in batches of 500

**Browsing:**
1. Paginated browse with filters: text search (across name, Messier, common names, identifiers), object type, constellation, magnitude range, minimum angular size, Messier-only toggle
2. Sorting by name, magnitude, angular size, or "best now" (visibility-based)
3. Observer latitude support: filter out objects that never rise, sort by current-season visibility
4. Can request specific objects by name list

**Thumbnails:**
1. Request for a catalog object thumbnail first checks a local cache (served as static files)
2. On cache miss, looks up the object's coordinates and angular size
3. Fetches a DSS2 color survey image from CDS hips2fits service, sized to the object's major axis
4. Caches the JPEG to disk for future requests (immutable cache headers)
5. Concurrency-limited to 25 simultaneous external fetches

**Target matching:**
1. Image tags (from plate solving) are matched against catalog object names
2. Matches are sorted by priority: Messier objects first, then brightest V-magnitude
3. The best match is set as the image's `targetName`

## Patterns Used

- Full-replace catalog loading (delete all, then bulk insert)
- Static file middleware for cached thumbnails with dynamic fallback handler
- Concurrency limiter (semaphore pattern) for external survey image fetches
- Cross-database search using `LOWER()` for case-insensitive matching on both SQLite and PostgreSQL
- Pure functions for CSV parsing and name normalization in the shared package (testable)

---

## Business Rules

- **Auto-load on startup:** If the catalog table is empty, the server automatically fetches and loads the catalog.
- **Name normalization:** All object names are normalized for consistent matching (zero-padded numbers stripped, spaces standardized).
- **Thumbnail sizing:** Survey thumbnails are sized based on the object's `majorAxis` with 1.5x padding, minimum 0.05 degrees FOV.
- **Filename sanitization:** Thumbnail cache filenames are sanitized to `[a-zA-Z0-9_-]` only to prevent path traversal.
- **Search minimum:** Autocomplete search requires at least 1 character.

---

## Edge Cases & Constraints

- Catalog load does a full delete-then-insert, which briefly leaves the table empty during reload
- The addendum CSV fetch failure is non-fatal — the catalog loads with NGC.csv only
- Objects without coordinates (RA/Dec) cannot have thumbnails generated
- Backfill-targets skips images that already have a `targetName` set

---

## Known Limitations

- No incremental catalog updates — reload replaces all data
- The `check-updates` endpoint exists but the actual comparison logic (checking GitHub commit dates) may not trigger automatic reloads
- Thumbnail generation depends on external CDS service availability
- Visibility calculation uses simplified constellation boundaries, not precise IAU polygon data
