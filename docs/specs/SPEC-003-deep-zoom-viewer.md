# SPEC-003: Deep Zoom Viewer

**Status:** Active
**Area:** image-management
**Date:** 2025-07-12

---

## Purpose

This feature provides a high-resolution, pannable, zoomable image viewer for astrophotography images. When plate-solving data is available, it overlays identified celestial objects as interactive annotations on the image, allowing users to explore what objects appear in their captures.

---

## Behavior

The deep zoom viewer is embedded within the image overlay. It uses OpenSeadragon for smooth pan/zoom interaction and can display plate-solving annotations as positioned overlays on the image.

### Entry Points

- Image overlay view — the viewer is the primary visual component
- `GET /api/images/:id/annotations` — Fetch plate-solving annotations and calibration data for overlay rendering

### Key Flows

1. Image loads in OpenSeadragon viewer with full pan/zoom capability
2. User can toggle between normal and expanded (full-screen) view modes
3. If the image is plate-solved, user can toggle annotation visibility
4. When annotations are enabled, the client fetches calibration data (RA, Dec, pixel scale, orientation) and annotation positions from stored plate-solving results
5. Annotations are rendered as positioned badges on the image using the calibration's WCS (World Coordinate System) data to map sky coordinates to pixel positions
6. Each annotation shows object names, type, and optional magnitude

---

## Patterns Used

- OpenSeadragon for deep-zoom image rendering
- Lazy-loaded annotations: only fetched when the user explicitly enables them (React Query with `enabled` flag)
- Pixel-coordinate mapping using plate-solving calibration data

---

## Business Rules

- **Annotations require plate solving:** The annotation toggle is only available for plate-solved images. Requesting annotations for a non-plate-solved image returns a 400 error.
- **Annotation data comes from stored results:** Annotations are served from the stored plate-solving job result, not re-fetched from astrometry.net.

---

## Edge Cases & Constraints

- Image dimensions from plate-solving results may be null, which affects annotation positioning accuracy
- Annotation pixel coordinates may use either `pixelx`/`pixely` or `pixelX`/`pixelY` naming (both are handled)
- Star-type annotations are included in the raw data but may be filtered for display relevance

---

## Known Limitations

- Images are loaded via Immich proxy URLs, not as deep-zoom tile pyramids — very large images may be slow to load
- No annotation filtering or search within the viewer
- No measurement tools (angular distance, field-of-view ruler)
